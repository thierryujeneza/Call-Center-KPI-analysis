# Call Center KPI Analysis

> End-to-end data analytics project on a 300-case call center dataset — covering Excel cleaning, SQL querying, Python automation, and Power BI dashboarding.

## Key finding
Extended calls consistently produced lower CSAT scores — confirming handle time efficiency as the primary driver of customer dissatisfaction, validated independently across SQL, Python, and Power BI.

## Tools used
`Microsoft Excel` `SQLite` `DB Browser` `Python 3` `pandas` `seaborn` `matplotlib` `Power BI Desktop`

---

## Project structure

| File | Description |
|------|-------------|
| `Call center data analysis.xlsx` | Cleaned dataset with COUNTIF duplicate detection, IFERROR imputation, and KPI formulas |
| `Call center.db` | SQLite database with 12 analytical queries |

---

## What was done

### 1. Excel — data cleaning
- Name standardisation with `PROPER()`
- Mean imputation for missing handle times using `IFERROR`
- Neutral midpoint fill for blank CSAT scores
- YES/NO flag columns for missing dates
- COUNTIF-based duplicate detection across all 300 rows
- Multi-dimensional analysis using `COUNTIFS`, `AVERAGEIFS`, `SUMIFS`, `MINIFS`, `MAXIFS`

### 2. SQL — multi-query analysis
- 12 queries in SQLite covering `SELECT`, `WHERE`, `GROUP BY`, `HAVING`, `JOIN`, subqueries, CTEs
- `RANK() OVER PARTITION BY` window functions
- `CASE WHEN` categorisation
- `CREATE VIEW` for reusable reporting

### 3. Python — automation & visualisation
- `pandas` groupby aggregations
- `seaborn` visualisations and correlation matrix
- Negative relationship confirmed: longer handle time → lower CSAT
- `openpyxl` multi-sheet Excel export

### 4. Power BI — interactive dashboard
- 15+ DAX measures including `CALCULATE`, `DIVIDE`, `RANKX`, `TOTALYTD`
- Conditional formatting and interactive slicers
- Dual-axis monthly trend chart
